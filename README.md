# Super-Crouton

A self-hosted Docker stack providing secure remote desktop access via an RDP gateway, backed by Keycloak for OIDC authentication, exposed publicly through Cloudflare Tunnels.

## Architecture

```
Internet
   │
   ▼  HTTPS (443)
 ┌──────────────────────────────┐
 │      Cloudflare Tunnel       │
 └──────┬───────────────────────┘
        │
   ┌────┴────────────────────────┐
   ▼                             ▼
auth.<your-domain>         rdp.<your-domain>
localhost:18080            localhost:19443
   │                             │
   ▼                             ▼
┌──────────┐              ┌───────────┐
│ Keycloak │◄─────────────│  rdpgw    │
│  + DB    │  OIDC/OpenID │           │
└──────────┘              └─────┬─────┘
                                │ RDP (3389)
                                ▼
                       your workstation(s)
```

TLS is terminated by Cloudflare — both services communicate over plain HTTP/S internally.

## Services

| Service | Image | Purpose | Local port |
|---|---|---|---|
| `keycloak` | `quay.io/keycloak/keycloak:24.0` | OIDC identity provider | `127.0.0.1:18080` |
| `keycloak-db` | `postgres:16` | Keycloak database | internal only |
| `rdpgw` | `bolkedebruin/rdpgw:latest` | RDP-over-HTTPS gateway | `127.0.0.1:19443` |

## Prerequisites

- Docker Engine 24+
- Docker Compose v2
- Python 3 (for the setup scripts — available by default on most Linux distros)
- A Cloudflare account with a domain whose DNS is managed by Cloudflare
- `cloudflared` installed and a tunnel created → see [docs/cloudflare-tunnels.md](docs/cloudflare-tunnels.md)

## Setup

### Step 1 — Keycloak

Run the interactive setup script to generate `keycloak/.env`:

```sh
./scripts/Step-1-Keycloak-setup.py
```

The script will ask for:

| Variable | Description |
|---|---|
| `KC_HOSTNAME` | Public hostname, e.g. `auth.example.com` (no `https://`) |
| `POSTGRES_PASSWORD` | PostgreSQL owner password |
| `KC_DB_PASSWORD` | Password Keycloak uses to connect to the DB (can match above) |
| `KC_BOOTSTRAP_ADMIN_PASSWORD` | Initial `admin` account password — used only on first startup |

Then start Keycloak:

```sh
cd keycloak && docker compose up -d
```

Once it's running, open `https://<KC_HOSTNAME>/admin` and:

1. Log in as `admin`
2. Create a realm (e.g. `my-realm`)
3. Go to **Clients → Create client**
   - **Client ID:** `rdpgw`
   - **Client authentication:** ON
   - **Standard flow:** enabled
   - **Valid redirect URIs:** `https://<rdp-gateway-hostname>/callback`
4. Copy the client secret from **Clients → rdpgw → Credentials**

### Step 2 — rdpgw

Run the interactive setup script to generate `rdpgw/.env` and `rdpgw/config/rdpgw.yaml`:

```sh
./scripts/Step-2-Rdpgw-setup.py
```

The script will ask for:

| Prompt | Description |
|---|---|
| Gateway hostname | Public hostname, e.g. `rdp.example.com` (no `https://`) |
| Local listen port | Host port Docker binds to (default: `19443`) |
| RDP hosts | One or more `hostname:port` targets, e.g. `gaming:3389` |
| OIDC provider URL | Full Keycloak realm URL, e.g. `https://auth.example.com/realms/my-realm` |
| OIDC client ID | Client ID from Keycloak (typically `rdpgw`) |
| OIDC client secret | Secret from Keycloak → Clients → rdpgw → Credentials |

All cryptographic session and token keys are **auto-generated** per deployment.

Then start rdpgw:

```sh
cd rdpgw && docker compose up -d
```

## Cloudflare Tunnels

Configure two public hostnames in your Cloudflare Tunnel:

| Public hostname | Service URL | Notes |
|---|---|---|
| `auth.<your-domain>` | `http://localhost:18080` | HTTP — Cloudflare handles TLS |
| `rdp.<your-domain>` | `https://localhost:19443` | HTTPS — rdpgw uses TLS internally |

> For the rdpgw tunnel, set **TLS verification** to **No TLS Verify** (or install a trusted cert) since rdpgw uses a self-signed cert by default.

## Connecting via RDP

Users access the gateway through a browser or directly via the Windows RDP client.

### Browser (recommended for first login)

Open `https://rdp.<your-domain>` — you will be redirected to Keycloak to log in, then offered an `.rdp` file to download and open.

### Windows Remote Desktop Connection (`mstsc.exe`)

1. Open **Remote Desktop Connection → Show Options → Advanced**
2. Under **Connect from anywhere**, click **Settings**
3. Select **Use these RD Gateway server settings**
4. Set **Server name** to `rdp.<your-domain>`
5. Connect to one of the hosts configured during setup (e.g. `gaming`)

### RDP Client Settings

`rdpgw/config/defaults.rdp` is served as the base `.rdp` template. It ships with:

- Multi-monitor support enabled
- Performance optimisations for LAN speeds (32-bit colour, font smoothing, no wallpaper/animations)
- Local audio playback and microphone passthrough

Edit `defaults.rdp` and restart the rdpgw container to change client-side defaults.

## Re-running Setup

The setup scripts can be re-run at any time. If an output file already exists, the script will ask before overwriting it. Security keys in `rdpgw.yaml` are regenerated on each run — restart the rdpgw container after re-running Step 2 to apply changes.

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| Keycloak admin UI stuck on "Loading" | `https://` prefix in `--hostname` flag | Hostname must be bare domain, no scheme |
| `Invalid parameter: redirect_uri` from Keycloak | `GatewayAddress` missing from `rdpgw.yaml` | Re-run Step-2 script |
| Cloudflare 502 on `rdp.<domain>` | Tunnel pointed at `http://` instead of `https://` | Change tunnel origin to `https://localhost:19443` |
| RDP connection slow or choppy | Default RDP settings | Edit `rdpgw/config/defaults.rdp` and restart rdpgw |
